Originally Answered: How can I get a source code for Android apps?

Refactored from StackOverflow (Answer by Prankul Garg)

Method 1

Using the online tool:

http://javadecompilers.com/apk

Method 2

Decoding step-by-step method:

Supposing your .apk file has the name myApp.apk

Step 1:

  1. Put your .apk file in a separate folder.
  2. Change its extension to .zip (e.g. myApp.apk => myApp.zip)

At this stage you are able to see drawables but not xml and java files.

Step 2:

  1. Extract this .zip file.
  2. Download dex2jar and extract it to the same folder.
  3. Move the classes.dex file into the dex2jar folder.
  4. Open command prompt and change directory to that folder. Then write d2j-dex2jar classes.dex (for Mac terminal or Ubuntu write ./d2j-dex2jar.sh classes.dex) and press enter. You now have the classes.dex.dex2jar file in the same folder.
  5. Download java decompiler, double click on jd-gui, click on open file, and open classes.dex.dex2jar file from that folder: now you get class files.
  6. Save all of these class files (In jd-gui, click File -> Save All Sources) by src name.

At this stage you get the java source but the .xml files are still unreadable, so continue.

Step 3:

  1. Put the .apk file which you want to decode in a new folder.
  2. Download the latest version of apktool AND apktool install window (both can be downloaded from the same link) and place them in the same folder created above.
  3. Run from the command line: apktool if framework-res.apk then apktool d myApp.apk (change myApp.apk to your .apk files name)

Now you get a file folder in that folder and can easily read the apk's .xml files.

Step 4:

Copy the contents of both folder (from Step 2 and Step 3) to the same location.

Thats all.
